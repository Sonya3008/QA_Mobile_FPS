**Bug 1 — Попадания не регистрируются при движении цели (Hitreg)**

ID: ..

Заголовок: Попадания не регистрируются при стрельбе по движущемуся противнику

Тип: Gameplay / Multiplayer

Приоритет: Critical

Серьёзность: Major

Окружение:

- Платформа: Android

- Сборка: Test Build v0.8.3

- Режим: Multiplayer

- Соединение: Wi-Fi, ping ~120 ms

Предусловия:

- Два игрока в одном матче

- Оба игрока активно перемещаются

Шаги воспроизведения:

1. Начать мультиплеерный матч

2. Дождаться, пока противник начнёт двигаться вбок

3. Навести прицел на модель противника

4. Произвести серию выстрелов

Фактический результат:

- Визуально пули попадают в модель

- Урон не наносится

- Индикаторы попаданий отсутствуют

Ожидаемый результат:

- Все визуальные попадания регистрируются

- Урон корректно рассчитывается

Дополнительно:

- Чаще воспроизводится при среднем и высоком ping

- Не наблюдается при стрельбе по стоящей цели

Вложения:

- скринкаст
- скриншоты
- логи

**Bug 3 — Фриз клиента при одновременных взрывах и стрельбе**

ID: ..

Заголовок: Кратковременный фриз клиента при интенсивном бое

Тип: Client Stability / Performance

Приоритет: High

Серьёзность: Major

Окружение:

- Платформа: Android

- Сборка: Test Build v0.8.3

- Режим: Multiplayer

- Устройства среднего сегмента

Предусловия:

- Матч с 6+ игроками

- Использование гранат

Шаги воспроизведения:

1. Начать матч с большим количеством игроков

2. Войти в зону активного боя

3. Во время стрельбы использовать гранату

4. Повторить несколько раз

Фактический результат:

- Клиент фризится на 2–4 секунды

- Потеря контроля над персонажем

- В некоторых случаях игрок погибает во время фриза

Ожидаемый результат:

- Игра остаётся отзывчивой

- FPS может снижаться, но без полной блокировки управления

Дополнительно:

Возможная причина: перегрузка эффектами

Вложения:

- скринкаст
- скриншоты
- логи
